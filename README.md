
# Cotton Collegiate Alumni Association Website
### Harness-Driven Static Website — Built with GitHub Copilot Agent Mode

---

## What This Project Is

A static website for the Cotton Collegiate Govt. H.S. School Alumni Association,
Guwahati, Assam. Established 1835.

This project uses **harness engineering** — a set of structured instruction files
that guide GitHub Copilot Agent Mode to build, verify, and maintain the website
autonomously. The developer provides knowledge and photos. The agent makes all
structural and design decisions.

---

## How It Works

```
You provide:       knowledge/*.txt files (school content in plain English)
                   assets/images/ (photos)
                   tickets/todo/ (requirements as ticket files)

Agent decides:     What pages to create
                   How to structure the navigation
                   Which photos to use where
                   How to fix its own issues

Agent produces:    All HTML pages
                   css/styles.css
                   js/main.js
                   knowledge/image-metadata.txt
                   build-log.txt
```

## Getting Started

### Step 1 — Add your content
Drop your plain text knowledge files into `knowledge/`.
Write naturally — no special formatting required.

### Step 2 — Add your photos
Drop all photos into `assets/images/`.
Organise into subfolders: `staff/`, `campus/`, `events/`.
Name files descriptively — e.g. `principal-jane-smith.jpg` not `IMG_4823.jpg`.

### Step 3 — Open in VS Code
```
code .
```

### Step 4 — Switch Copilot to Agent Mode
1. Open the Copilot Chat panel (sidebar)
2. Click the mode dropdown at the bottom of the chat panel
3. Select **Agent**

### Step 5 — Run the build prompt
Copy this prompt from `PROMPTS.md` and paste it into the Agent Mode chat:

```
Follow the skill in .github/skills/build-and-verify.md completely.
Run all phases from start to finish without stopping for approval.
Build the full site, verify it, fix all issues found, and report
back with a summary when complete.
```

### Step 6 — Review the result
Open your browser and go to:
```
http://localhost:8080
```

## Adding New Requirements

Create a ticket file in `tickets/todo/` using this format:

```
Ticket: CCAA-001
Title: [short description of the requirement]
Priority: High / Medium / Low
Description:
[what you want in plain English]

Acceptance Criteria:
- [specific thing that must be true when done]
- [another specific thing]
- [another specific thing]
```

Then paste this prompt into Agent Mode:

```
Check tickets/todo/ for pending tickets and start work.
Follow the ticket workflow in .github/skills/build-and-verify.md.
Pick the highest priority ticket, implement it, verify it,
self-correct any issues, and report back when done.
```

The agent will:
- Pick up the highest priority ticket
- Move it to `tickets/in-progress/`
- Implement the requirement
- Verify and self-correct
- Move it to `tickets/done/` with a completion note
- Report back to you

## What the Agent Manages Automatically

You never need to manually edit these files:

| File | What the agent does with it |
|---|---|
| `knowledge/image-metadata.txt` | Scans all photos and writes this file |
| `css/styles.css` | Writes and updates all styles |
| `js/main.js` | Writes and updates all JavaScript |
| `build-log.txt` | Logs all build decisions, issues, and fixes |
| All `*.html` files | Creates and updates all pages |
| `tickets/in-progress/` | Moves tickets here while working |
| `tickets/done/` | Moves tickets here when complete |

No frameworks. No npm. No build tools. No dependencies.

---

## Important Rules

- Never edit `css/styles.css`, `js/main.js`, or any `.html` file manually.
  Always use the agent to make changes so the harness rules are followed.
- Never edit `knowledge/image-metadata.txt` manually.
  Always ask the agent to scan and update it.
- Always name photo files descriptively before adding them to `assets/images/`.
- Always write ticket files in `tickets/todo/` — never give requirements verbally
  without a ticket file, as the agent will not track or log them.
